Checked state test case

The examples on this page use the <input> element with type="checkbox" to test the way the checked and indeterminate states are reported by screen readers.

Unchecked state

Example code


<input type="checkbox" id="unchecked">
<label for="unchecked">Lorem ipsum</label>

Checked state

Example code


<input type="checkbox" id="checked" checked>
<label for="checked">Dolor</label>

Indeterminate state

Example code


<input type="checkbox" id="indeterminate">
<label for="indeterminate">Sit amet</label>



document.getElementById("indeterminate").indeterminate="true";

Notes

The checked/unchecked state is exposed through the accessibility API in Chrome, Firefox, IE and Safari. The indeterminate state is exposed in Chrome and Firefox, but not IE or Safari.

Jaws and NVDA announce all three states correctly in Chrome and Firefox on Windows. Window-Eyes announces all three states correctly in Firefox and IE on Windows.

VoiceOver announces the checked/unchecked states correctly in Safari on OS X and iOS, but not the indeterminate state. TalkBack announces the checked/unchecked states correctly in Chrome on Android, but not the indeterminate state.
